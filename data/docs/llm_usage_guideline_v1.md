# LLM Usage Guideline (v1.0)

## Allowed use
- Drafting internal emails and summaries (non-sensitive content).
- Code assistance for non-confidential projects.
- Document Q&A when documents are approved for the assistant.

## Disallowed use
- Inputting **Restricted** data into external LLM services without an approved contract and safeguards.
- Generating final legal, medical, or compliance decisions without human review.

## Prompting rules
- Ask for sources and cite policy sections.
- Do not request personal data unless required and approved.
- Use minimal necessary context.

## Human-in-the-loop
- High impact outputs require review:
  - customer communications
  - policy interpretations affecting compliance
  - incident severity decisions
