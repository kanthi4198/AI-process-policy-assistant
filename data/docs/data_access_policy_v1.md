# Data Access Policy (v1.0)

## Purpose
This policy defines how employees request, obtain, and review access to internal datasets.

## Access levels
- **Public**: externally shareable, no approval required.
- **Internal**: available to employees with business need.
- **Restricted**: sensitive; requires formal approval and logging.

## Request process
1. Identify dataset name and business purpose.
2. Submit request with:
   - data owner / domain
   - intended use
   - retention duration
   - whether personal data is involved
3. Approval requirements:
   - Internal: data owner approval
   - Restricted: data owner + Security Office + Data Protection review (if personal data)

## Review & expiry
- Access to **Restricted** data expires after **90 days** unless renewed.
- Reviews occur quarterly for high-risk datasets.

## Prohibited actions
- Copying Restricted data to personal devices.
- Sharing credentials or access tokens.
